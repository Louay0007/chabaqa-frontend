# Arabic Product Glossary

## Purpose
Authoritative terminology for Arabic UX copy. Use these terms consistently across all pages and messages.

| English Term | Arabic Term | Usage Note |
|---|---|---|
| Community | مجتمع | Default label for community entity. |
| Creator | صانع المحتوى | Prefer this over generic "منشئ" in product UI. |
| Dashboard | لوحة التحكم | Use for all admin/creator main dashboards. |
| Course | دورة | Keep concise in tabs/cards. |
| Challenge | تحدي | Use singular for item, plural `تحديات`. |
| Product | منتج | For digital/physical listings. |
| Session (1:1) | جلسة (1:1) | Keep `(1:1)` where needed for clarity. |
| Event | فعالية | Plural: `فعاليات`. |
| Checkout | إتمام الشراء | Also acceptable: `الدفع` for button contexts. |
| Membership | عضوية | For paid access context. |
| Join | انضمام | Button style: `انضم الآن`. |
| Sign In | تسجيل الدخول | Standard auth action. |
| Sign Up | إنشاء حساب | Prefer over transliteration. |
| Verify Email | تأكيد البريد الإلكتروني | Used in registration verification flow. |
| Reset Password | إعادة تعيين كلمة المرور | Keep full phrase; avoid abbreviations. |
| Featured | مميز | Badge/status context. |
| Verified | موثق | Badge/status context. |
| Active | نشط | Status context. |
| Pending | قيد الانتظار | Status context. |
| Rejected | مرفوض | Status context. |
| Save Settings | حفظ الإعدادات | Settings pages/actions. |
| Analytics | التحليلات | Charts/reporting context. |
| Revenue | الإيرادات | Financial context. |
| Payout | مدفوعات السحب | Creator/admin finance context. |
| Content Moderation | مراجعة المحتوى | Admin moderation context. |

## Style Notes
- Keep imperative buttons short and direct.
- Prefer clear MSA wording over dialect in core actions.
- Preserve technical tokens where needed (`1:1`, `VIP`, `URL`).
- Do not auto-translate brand name `Chabaqa` unless context requires Arabic rendering `شبكة`.

## Conflict Resolution
When two teams propose different Arabic wording for the same term:
1. Follow this glossary by default.
2. Raise a glossary change PR with examples and affected keys.
3. Approve with product/content owner before rollout.
